Subjects

Read and write (1/4)

A Subject is both an Observable and an observer: 


    const subject = new Subject();

    // Observable:
    subject.subscribe({
      next:     (value) =>  { console.log(value);       },
      complete: ()      =>  { console.log('complete');  },
    });
    
    // observer:
    subject.next(1);     // 1
    subject.next(2);     // 2
    subject.complete();  // 'complete'

BehaviorSubjects (2/4)

A BehaviorSubject sends the "current" value to subscribers:


      const subject = new BehaviorSubject(0);
      
      subject.subscribe((value) => { console.log(`A: ${value}`); });
      subject.next(1);
      subject.next(2);
      
      subject.subscribe((value) => { console.log(`B: ${value}`); });
      subject.next(3);

      // A: 0
      // A: 1
      // A: 2
      // B: 2
      // A: 3
      // B: 3

ReplaySubjects (3/4)

A ReplaySubject sends the most recent n values to subscribers: 


      const subject = new ReplaySubject(2);
      
      subject.subscribe((value) => { console.log(`A: ${value}`); });
      subject.next(1);
      subject.next(2);
      subject.next(3);
      
      subject.subscribe((value) => { console.log(`B: ${value}`); });
      subject.next(4);
      
      // A: 1
      // A: 2
      // A: 3
      // B: 2
      // B: 3
      // A: 4
      // B: 4

AsyncSubjects (4/4)

An AsyncSubject sends the last value before completion to subscribers:


      const subject = new AsyncSubject();
      
      subject.subscribe((value) => { console.log(`A: ${value}`); });
      
      subject.next(1);
      subject.next(2);
      
      subject.subscribe((value) => { console.log(`B: ${value}`); });
      
      subject.next(3);
      subject.complete();
      
      // A: 3
      // B: 3
