Arrow functions (2/4)

  • When you need the return value of an arrow function, you can omit { and }: 
    
          const a = () => {
            return 1;
          }
          a();  // 1
        
    
          const a = () => 1;


          a();  // 1
  • If you want to return an object, use parentheses: 
    
        const a = () => ({ a: 1 });
        a();  // { a: 1 }

First-class (3/4)

  • JS functions are first-class. They can be:
    • stored in a variable 
      
            const a = () => 1;
            const b = a;
            b;                      // () => 1
            b();                    // 1
    • passed as arguments to functions 
      
            const a = () => 1;
            const b = (fn) => fn();
            b(a);                   // 1
    • returned from functions 
      
            const a = () => 1;
            const b = (fn) => fn;
            b(a);                   // () => 1
            b(a)();                 // 1

Closures (4/4)

A closure is when an inner function has access to its surrounding function(s) scope, even after the surrounding function(s) have returned: 


      const a = 1;

      const parent = () => {
        const b = 2;

        return () => {
          const c = 3;
          return [a, b, c];
        }
      }

      const child = parent();
      child();  // [1, 2, 3]

Closures enable:

  • data privacy
  • partial application: 
    
            const add = a => b => a + b;
            const add1 = add(1);

            add1(2);  // 3
