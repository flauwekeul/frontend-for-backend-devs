Type coercion (1/4)

  • JS allows implicit conversion of types: 
    
        '2' * '3';    // 6
        'a' + 1;      // 'a1'
        true + false; // 1
  • Any type in JS can be converted: 
    
        {}+[]+{}+[1]; // '0[object Object]1' ☹️

Type coercion (2/4)

  • All operators (except ===) trigger type coercion: 
    
        1 >= '1'; // true
        1 + '1';  // '11'
        1 / '1';  // 1
  • ⚠️ Logical operators don't return booleans, but one of their operands instead: 
    
        0 && 1;         // 0
        0 || 1;         // 1
        ES2020
        0 ?? 1;         // 0
        null ?? 1;      // 1
        undefined ?? 1; // 1

Type coercion (3/4)

  • Explicit type coercion is possible too: 
    
        String(1);      // '1'
        String(true);   // 'true'
        Boolean(1);     // true
        Boolean('');    // false
        Number('1');    // 1
        Number(false);  // 0
  • But implicit type coercion has its advantages: 
    
          // please don't do this
          if (Boolean(2)) { … }
          Boolean(2) || Boolean('a');
        
    
          // keep it simple
          if (2) { … }
          2 || 'a';

Type coercion (4/4)

  • In JS these values are falsy: 
    
        Boolean('');         // false
        Boolean(0);          // false
        Boolean(NaN);        // false
        Boolean(null);       // false
        Boolean(undefined);  // false
        Boolean(false);      // false
  • Everything else is truthy: 
    
        Boolean('a');      // true
        Boolean(1);        // true
        Boolean({});       // true
        Boolean([]);       // true
